## overview
- classes and constructors (class, def, self, etc)
- self parameter
- class/objects *instance types*
	- class attribute: shared among all attributes (variable)
	- instance attribute: unique to each instance (self.variable)
- class customization (print overload)
- **operator overloading**
	- `__lt__(self, other)`
	- `__le__(self, other)`
	- `__gt__(self, other)`
	- `__ge__(self, other)`
	- `__eq__(self, other)`
	- `__ne__(self, other)`
	- if you do not know these i suggest you quit
- uml class diagram
	- class name on top, then attributes, then instance methods
	- `+` for public and `-` for private
		- as naming convention, private member names start with underscore.
- numeric type classes
	- `__add__(self, other)` +
	- `__sub__(self, other)` -
	- `__mul__(self, other)` *
	- `__truediv__(self, other)` /
	- `__floordiv__(self, other)` //
	- `__mod__(self, other)` %
	- `__pow__(self, other)` **
	- `__and__(self, other)` and
	- `__or__(self, other)` or
		- `__abs__(self, other)` abs()
	- `__int__(self, other)` int()
	- `__float__(self, other)` float()
- object composition
	- one object is made up of other (smaller) objects
- inheritance, inheritance tree, inheritance types (single, multiple, multilevel, hierarchical, hybrid)
	- class method overriding
	- diamond problem, **method resolution order**
- memory allocation, garbage collection (once deallocated)
