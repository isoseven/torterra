## recursion
- **recursion**: defining an object in terms of itself
- recursion can be used to **define sequences**
	- previously sequences are defined as a specific formula.
		- *ex.* $a_n = 2^n \textrm { for } n = 0, 1, 2, \ldots$ 
		- this sequence can also be defined by giving the **first term** of the sequence ($a_0 = 1$), and **a rule** for finding the term of the sequence for the previous one ($a_{n+1} = 2a_n \textrm{ for } n = 0, 1, 2, \ldots$) 
- when defining a set recursively, we
	1. specify the initial elements in a **basis step**
	2. provide a rule for constructing new elements (**recursive step**)
	- *ex.* arithmetic or geometric sequence
		- arithmetic: $a_0 = a, a_n = d + a_{n-1} \textrm{ for } n \ge 1$ 
		- geometric: $a_0 = a, a_n = r \cdot a_{n-1} \textrm{ for } n \ge 1$

## recurrence relations
- *ex.* fibonacci sequence
	- initial values $f_0 = 0, f_1 = 1$
	- recursive step $f_n = f_{n-1} + f_{n-2} \textrm{ for } n \ge 2$
-  *ex.* sequence $b_n$
	- $b_0 = 1, b_1 = 2, b_n = b_{n-1} \cdot b_{n-2}+ 4 \textrm{ for } n \ge 2$
	- $b_2 = 6$
	- $b_3 = 16$
- sometimes we can **reduce** the solution with a particular set of inputs to a solution of the same problem with smaller input values.
	- when this reduction is possible, the solution to the original problem can be found with a **sequence of reductions** until the problem has been reduced to the **initial case**

## recursive algorithms
### $n!$
- recursive factorial function:
	- $f(n) = n! \textrm{ for } n \ge 0$
	- $f(n) = n! = n \cdot (n-1) \cdot (n-2) \ldots 1$
- recursive form:
	- $f(0) = 1$
	  $f(n) = n \cdot f(n-1)$

### $a^n$
- pseudo code

  ```pseudo
  procedure power(a: nonzero real number, n: nonnegative integer)
	  if n = 0 then
		  return power(a, n) = 1
	  else 
		  return power(a, n) = a * power(a, n-1)		  
  ```
## (recursive) search algorithms
### linear search
```
procedure linear-search(i, j, x: integers, 1 <= i <= j <= n)
	if i > j then location := 0
	else if a_i = x then location := i
	else linear-search(i+1, j, x)
```
### binary search
```
procedure binary-search(i, j, x: integers, 1 <= i <= j <= n)
	if i > j then location := 0
	else
		m = floor((i + j) / 2)
		if x = a_m then location := m
		else if (x < a_m AND i < m) then binary-search(i, m-1, x)
		else if (x > a_m AND j > m) then binary-search(m+1, j, x)
		
```
### merge sort
```
procedure merge(L_1, L_2: sorted lists)
	L := empty list
	while L_1 and L_2 are both nonempty
	begin
		remove smaller of first element of L_1 and L_2 from the list it is in and put it at the right end of L
		if removal of this element makes one list empty then
			remove all elements from the other list and append them to L
	end
```
